⚕️ What is RAD‑XR?

RAD‑XR is an enterprise‑grade DICOM receiver and medical imaging automation platform built for radiology departments, imaging centers, and healthcare IT teams. It listens for incoming DICOM studies over the network, automatically groups images by patient, generates beautiful multi‑page PDF reports, and delivers them instantly via Telegram and WhatsApp Business.

Designed for high‑volume environments, RAD‑XR handles compressed DICOM (JPEG, JPEG2000, RLE, etc.), supports batching, maintains a full SQLite index, and offers a modern GUI for live monitoring and configuration. It's the bridge between your PACS and your clinical workflow.

✨ Core Features

📡

DICOM Server (SCP)

Listens on any IP/port, accepts C‑STORE and C‑ECHO, supports all standard SOP classes and compressed transfer syntaxes.

📄

Multi‑Page PDF Reports

Combines multiple DICOM images into a single PDF with patient metadata, footer image support, and beautiful layout.

🤖

Telegram Bot Integration

Auto‑sends PDF reports to admin users and groups. Users can request reports by Patient ID + Name.

💬

WhatsApp Business API

Sends PDF reports directly to WhatsApp using the phone number from the Accession Number. Credits‑based usage.

🗂️

Smart Patient Batching

Groups incoming DICOM files by Patient ID or Name. Waits for a configurable time to collect all images of a study.

📁

Folder Monitoring (Archive)

Watches the Archive folder for new DICOM files added by external systems and processes them automatically.

🔍

Patient Index & Cache

SQLite database indexes every DICOM file. PDF reports are cached and can be re‑sent on demand.

💳

Credits System

Separate credit balances for Telegram and WhatsApp. Each send consumes one credit. Recharge via GUI or Telegram.

🛡️

Admin & Group Management

Add/remove admin users and Telegram groups. Master user has full control. Users can request reports.

⚙️

Full GUI Dashboard

Built with Tkinter — live monitor, network config, credits, console log, and password‑protected settings.

⚙️ How It Works

STEP 1

Receive DICOM

DICOM files arrive via C‑STORE (network) or are dropped into the Archive folder. All standard modalities supported.

STEP 2

Batch by Patient

Files are grouped by Patient ID or Name. A timer waits for additional images of the same study to arrive.

STEP 3

Generate PDF

All images are assembled into a single multi‑page PDF with patient metadata on each page.

STEP 4

Deliver & Archive

The PDF is sent to all configured Telegram admins/groups and, if Accession Number is present, via WhatsApp.
