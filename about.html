About RAD‑XR

The complete DICOM receiver, PDF report generator, and multi‑platform medical imaging gateway.

⚕️ What is RAD‑XR?

RAD‑XR is an enterprise‑grade DICOM receiver and medical imaging automation platform built for radiology departments, imaging centers, and healthcare IT teams. It listens for incoming DICOM studies over the network, automatically groups images by patient, generates beautiful multi‑page PDF reports, and delivers them instantly via Telegram and WhatsApp Business.

Designed for high‑volume environments, RAD‑XR handles compressed DICOM (JPEG, JPEG2000, RLE, etc.), supports batching, maintains a full SQLite index, and offers a modern GUI for live monitoring and configuration. It’s the bridge between your PACS and your clinical workflow.

✨ Core Features

📡

DICOM Server (SCP)

Listens on any IP/port, accepts C‑STORE and C‑ECHO, supports all standard SOP classes and compressed transfer syntaxes (JPEG, JPEG2000, RLE).

📄

Multi‑Page PDF Reports

Combines multiple DICOM images (or multi‑frame) into a single PDF with patient metadata, footer image support, and beautiful layout — one report per patient.

🤖

Telegram Bot Integration

Auto‑sends PDF reports to admin users and groups. Users can request reports by Patient ID + Name. Full command set for admin: add/remove users, groups, broadcast, credits.

💬

WhatsApp Business API

Sends PDF reports directly to WhatsApp using the phone number from the Accession Number. Supports media upload and caption formatting. Credits‑based usage.

🗂️

Smart Patient Batching

Groups incoming DICOM files by Patient ID or Name. Waits for a configurable time (default 15s) to collect all images of a study before generating a single combined PDF.

📁

Folder Monitoring (Archive)

Watches the Archive folder for new DICOM files added by external systems (e.g., PACS export) and processes them automatically through the same pipeline.

🔍

Patient Index & Cache

SQLite database indexes every DICOM file by Patient ID, Name, and Accession Number. PDF reports are cached and can be re‑sent on demand — no need to regenerate.

💳

Credits System

Separate credit balances for Telegram and WhatsApp. Each send consumes one credit. Admins can recharge via Telegram commands or the GUI. Prevents runaway usage.

🛡️

Admin & Group Management

Add/remove admin users (auto‑send recipients) and Telegram groups. Master user has full control. Users can also be granted access to request reports.

⚙️

Full GUI Dashboard

Built with Tkinter — live monitor shows incoming files, status, network config, credits, and console log. Configuration tab for all settings with password protection.

⚙️ How It Works

STEP 1

Receive DICOM

DICOM files arrive via C‑STORE (network) or are dropped into the Archive folder. The server accepts all standard modalities (CT, MR, US, CR, DX, etc.).

STEP 2

Batch by Patient

Files are grouped by Patient ID or Name. A timer (configurable) waits for additional images of the same study to arrive before processing.

STEP 3

Generate PDF

All images (or frames) are assembled into a single multi‑page PDF. Metadata (Patient Name, ID, Sex, Study Date, Accession, Modality) is printed on each page.

STEP 4

Deliver & Archive

The PDF is sent to all configured Telegram admins/groups and, if Accession Number is present, to the patient via WhatsApp. DICOM files are deleted on success.
